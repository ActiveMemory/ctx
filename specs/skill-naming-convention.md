# Spec: Skill Naming Convention — ctx-$domain-$action

## Problem

Skill names use inconsistent patterns: some are `$action-$domain`
(`ctx-add-task`, `ctx-check-links`, `ctx-sanitize-permissions`),
some are `$domain-$action` (`ctx-journal-enrich`, `ctx-skill-audit`),
and one is a noun (`ctx-skill-creator`). This inconsistency makes
skills harder to discover and predict.

## Convention

All skills follow **`ctx-$domain-$action`** format:

- Domain first: groups skills by what they operate on
- Action second: describes what the skill does
- Singular domain always: `ctx-permission-sanitize` not
  `ctx-permissions-sanitize`
- Mirrors CLI subcommand structure: `ctx task complete`,
  `ctx pad add`, `ctx journal source`

## Surfaces to Update Per Rename

Each rename touches:

1. **Skill directory**: `internal/assets/claude/skills/$old/` →
   `internal/assets/claude/skills/$new/`
2. **SKILL.md `name:` frontmatter**: update the name field
3. **SKILL.md `description:`**: update if it references the old name
4. **Cross-references in other skills**: grep all SKILL.md files for
   `/ctx-old-name` and update to `/ctx-new-name`
5. **Copilot skill mapping**: `internal/assets/copilot/skills/` if
   mirrored there
6. **Documentation**: all `docs/**/*.md` references to `/ctx-old-name`
7. **Blog posts**: leave as-is (historical)
8. **AGENT_PLAYBOOK.md**: update conversational trigger table
9. **CLAUDE.md**: update if referenced
10. **Test files**: update any skill name assertions
11. **Site search index**: regenerated by site build
12. **Skill registry/catalog**: if skills are listed in a catalog file

## Implementation Order

Rename one skill at a time to keep diffs reviewable. Order by
dependency (rename skills that are cross-referenced by others last):

1. `ctx-add-task` → `ctx-task-add` (most referenced)
2. `ctx-add-decision` → `ctx-decision-add`
3. `ctx-add-learning` → `ctx-learning-add`
4. `ctx-add-convention` → `ctx-convention-add`
5. `ctx-skill-creator` → `ctx-skill-create`
6. `ctx-sanitize-permissions` → `ctx-permission-sanitize`
7. `ctx-import-plans` → `ctx-plan-import`
8. `ctx-check-links` → `ctx-link-check`
